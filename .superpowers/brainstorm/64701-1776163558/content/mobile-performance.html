Design: Floating Interest Objects

Section 4 of 4 — Mobile & Performance

Mobile Behavior

Interaction
  • No hover state on mobile — tap to discover directly
  • Interest card appears as bottom sheet (same pattern as Saturn project cards)
  • Swipe down or tap outside to dismiss
  • Discovery counter moves to top-right to avoid thumb zone conflict
Visibility
  • Objects slightly larger on mobile (easier tap targets)
  • Positioned closer to center of camera path (smaller viewport)
  • Glow aura slightly brighter so they stand out on small screens
  • Reduce bob amplitude so they don't drift off-screen

Performance Budget

Sprites 9 quads — negligible draw calls (could batch into one if needed) Textures 9 emoji textures generated via CanvasTexture at init — no network requests Raycaster Only fires on click/tap, not every frame — zero ongoing cost Animations Bob + rotation in useFrame — trivial sin/cos per sprite per frame GPU tier scaling Low-end devices: disable glow aura, reduce to static sprites localStorage Single JSON key for discovered array — read once on mount

Configurability (Future-Proofing)

  • interests.js — single data file is the only thing to edit. Each entry is:
{
  id: "badminton",
  emoji: "🏸",
  name: "Badminton",
  description: "District Level, Age 16",
  position: [x, y, z],  // 3D coordinates
  color: "#FFD700",      // accent color when discovered
}
  • Add a new interest → add one object to the array. That's it.
  • Remove one → delete the object. Counter auto-adjusts.
  • Reposition → change the [x, y, z] values.
  • Counter denominator derived from array length — never hardcoded.
